## Introduction

Formémon's codebase is set up in a monorepo via [Turborepo](https://turbo.build/repo) and is fully [open-source on GitHub](https://git.new/Formémon).

Here's the monorepo structure:

```
apps
├── api
├── dashboard
├── docs
├── engine
├── mobile
├── website
packages
├── documents
├── email
├── events
├── import
├── inbox
├── jobs
├── kv
├── location
├── notification
├── supabase
├── tsconfig
├── ui
├── utils
```

The `apps` directory contains the code for:

- `api`: The backend for Formémon (supabase).
- `dashboard`: Formémon's application ([app.Formémon.ai](https://app.Formémon.ai)).
- `docs`: Formémon's documentation site ([Formémon.ai/docs](https://Formémon.ai/docs)).
- `engine`: Formémon's bank engine ([Formémon.ai/engine](https://Formémon.ai/engine)).
- `mobile`: Formémon's mobile app.
- `website`: Formémon's website app ([Formémon.ai](https://Formémon.ai)).

The `packages` directory contains the code for:

- `documents`: OCR for reciepts.
- `events`: Analytics library.
- `import`: CSV-importer.
- `inbox`: Inbox utils.
- `jobs`: Background jobs.
- `kv`: Key value storage client.
- `location`: Location related utils.
- `notification`: Notifications client.
- `supabase`: Queries and client.
- `tsconfig`: The TypeScript configuration for Formémon's web app.
- `ui`: UI component library.
- `utils`: A collection of utility functions and constants used across Formémon's codebase.

## Running Formémon locally

To run Formémon locally, you'll need to set up the following:

- A [Supabase](https://supabase.com/) account
- An [Upstash](https://upstash.com/) account

## Step 1: Local setup

First, you'll need to clone the Formémon repo and install the dependencies.

<Steps>

  <Step title="Clone the repo">

First, clone the [Formémon.ai repo](https://d.to/github) into a public GitHub repository.

    ```bash Terminal
    git clone https://github.com/Formémon-ai/Formémon.git
    ```

  </Step>

  <Step title="Install dependencies">

    Run the following command to install the dependencies:

    ```bash Terminal
    bun i
    ```

  </Step>

</Steps>
